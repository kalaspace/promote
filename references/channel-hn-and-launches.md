# Hacker News + Launch Days (Product Hunt, Indie Hackers, OSS launches)

> **Channel scope** : ce framework couvre Hacker News (Show HN, Ask HN, posts), Product Hunt launch days, Indie Hackers community, et open-source launch days (GitHub release + cross-post). **Ne couvre PAS** : paid ads sur ces plateformes (HN n'en a pas, PH job board = autre chose), ni ongoing community management long terme (→ canal-spécifique strategist).
> **Why framework, not operator** : la doctrine est **convergente** sur ces canaux (open-source day 1, founder voice authentique, builder credibility), portée par plusieurs voix (Marc Lou, Sahil Lavingia, Pieter Levels, Ryan Hoover, Patrick McKenzie), mais aucune n'est assez singulière pour mériter un operator distillé. Le pattern est plus fort que les personnes.
> **Used by** : `promote-channel-strategist` quand le launch event est dans le mix (Show HN, PH launch, GitHub release annonce). Particulièrement pertinent pour dev tools, OSS, indie SaaS, hardware.

## Quand consulter ce framework

**Fits** :
- Produit avec **angle technique** ou **builder story** convaincante (dev tool, OSS lib, AI utility, hardware).
- Founder/team **présente sur place** le jour du launch (réponses dans les 5 minutes pendant la fenêtre frontpage).
- Une **vraie nouveauté** à montrer (pas un repackaging — HN sniff la repackaging immédiatement).
- Fenêtre 2026 : ICP = devs, founders, technologists, indie hackers.

**Anti-fits** :
- Produit purement marketing/sales (CRM saaS générique → HN downvote, PH plateau).
- Founder absent jour J (pas de réponses live = mort frontpage en 30 min).
- Lancement répété sur le même produit sans nouveauté (HN tracke les "relaunches" et les pénalise).
- Cible enterprise C-level (PH = early adopters, HN = builders, ni l'un ni l'autre = enterprise buyer).

## Tactiques 2026

### 1. HN posting times et title psychology

- **Best slots 2026** : mardi-jeudi, 9h-11h EST. Évite weekend (faible volume voters), évite vendredi (drop algo). Évite dates US holidays (Thanksgiving, July 4, Christmas).
- **Title HN psychology** :
  - "Show HN: [thing] — [one-line value]" : format canonique.
  - Pas de marketing adjectives ("revolutionary", "powerful", "best"). HN downvote.
  - Inclure technologie/stack si pertinent ("Show HN: A Rust CLI for X") — augmente CTR builders.
  - Sub-50 chars idéal pour mobile + frontpage display.

### 2. Show HN vs Ask HN mechanics

- **Show HN** : tu montres quelque chose de fonctionnel. **Doit être live + free to try sans login** (idéal). Sinon HN frustré.
- **Ask HN** : tu poses une vraie question à la communauté. Plus organique, mais ne sert pas le launch (reserve pour problem discovery).
- **Tell HN** : opinion/announce. Faible reach, n'utilise que pour annonces majeures (acquisition, sunset).

### 3. La fenêtre frontpage 90 minutes

- HN algorithme 2026 : ton post a ~90 minutes après initial visibility pour atteindre la frontpage. Si tu n'es pas page 1 à 90 min, tu n'y arriveras plus.
- **Pendant ces 90 min** :
  - Réponds à CHAQUE comment substantiel sous 5 min.
  - Comments génèrent de l'engagement → boost score → frontpage.
  - Sois **technique, précis, humble**. HN punit le marketing, récompense le builder voice.
- Après 4h sur frontpage, traffic spike massif (5k-50k visiteurs uniques selon position).

### 4. Comments-driven traction (le vrai jeu HN)

Les upvotes ne suffisent pas. Le ranking algo HN 2026 pondère lourdement le **comment count + author engagement**.

- Pre-anticipe 10-15 questions techniques courantes (pricing, OSS license, comparison à X, your stack, perf benchmarks, security).
- Réponds avec **détail réel** (pas "great question!"). HN respecte la profondeur.
- Si quelqu'un trouve un bug live, fixe-le pendant le thread et commente "fixed in commit X". Légendaire move.

### 5. Product Hunt launch day playbook

Pattern dominant 2026 (Sahil Lavingia, Marc Lou, Ryan Hoover-style) :

- **Pre-launch 2 semaines** : build une notify-list (200-500 people minimum). Envoie heads-up email J-1.
- **Hunter** : recrute un hunter respecté (Chris Messina, Kevin William David, Marc Lou) idéalement. Ou auto-hunt si tu as ≥50 followers PH.
- **Launch 12:01 AM PST** (fuseau PH). Premiers 30 min = critiques pour ranking.
- **Tagline** : ultra-tight, 60 chars max, value claire. "AI agents that ship code" > "Revolutionary AI platform".
- **Galerie** : 6 visuels — hero shot, key feature 1, feature 2, social proof, founder photo, CTA. GIF welcomed.
- **Comments** : founder répond toutes les questions pendant 24h. Idem HN — engagement → ranking.
- **Day-of email** : 2 emails à ta liste (matin, milieu de journée). Pas plus.
- **#1 Product of the Day** = trophée + 2-5k visites + relais Twitter/LinkedIn organique.

### 6. Pre-launch hunter network

- Builder relationships **avant** d'avoir besoin. Comment sur 20 launches PH en upstream (sans demander rien).
- Le jour J, tu actives la dette. 5-10 hunter friends qui upvote-comment dans les 30 premières minutes = boost initial décisif.
- Anti-pattern : DM blast à des hunters le jour du launch. Méprisé. Construis la relation 3+ mois en amont.

### 7. Indie Hackers community gradients

IH 2026 = community plus que channel. Use cases :

- **Milestones posts** ($1k MRR, $10k MRR) : récompense forte si chiffres réels + post-mortem honnête.
- **Build in public** : transparent revenue share + lessons. Build trust durable.
- **Founder stories** : longform, vulnerable, technical. Marie-Claire Dean / Tony Dinh / Marc Lou patterns.

### 8. Open-source launch day mechanics

Si tu lances un OSS :

- **GitHub release** = source of truth. README parfait, demo GIF, install one-liner.
- **Cross-post simultané** : Show HN + r/programming (si fit) + Twitter/X thread + Hacker Newsletter inclusion (si tu connais l'editor).
- **Stars trajectory J+7** : 100-500 stars = succès indie. 1k+ = viral. Cible >10 contributors externes en 30 jours.

## Cadence + format recommandé

- **Volume** : 1 launch majeur tous les 3-6 mois max. Sur-launching = fatigue de l'audience.
- **Formats prioritaires** : (a) Show HN avec live demo ; (b) PH launch avec gallery + GIF ; (c) IH milestone post avec vraies metrics ; (d) GitHub release annonce cross-canal.
- **Anti-formats** : "we're back!" relaunches sans nouveauté, marketing-heavy copy sur HN, generic PH taglines.

## Anti-patterns 2026

1. **Marketing voice sur HN** : "Revolutionary AI-powered platform that disrupts..." → downvote within 10 min. HN demande builder voice (technique, humble, précis).
2. **Vote manipulation** : Slack/Discord blast pour upvotes coordonnés. HN détecte le pattern (votes en cluster temps + IP) et flag/penalize.
3. **Founder absent jour J** : tu poses ton link et tu disparais. HN/PH algo penalize, traffic chute, et tu rates les questions critiques.
4. **PH launch sans pre-launch** : tu lances cold sans liste, sans hunter, sans warm comments. Score reste bas, ranking < #5, peu de traffic.
5. **Tagline marketing-puffy** : "Revolutionary platform that empowers businesses" → 0 conversion. PH veut value claire en 60 chars.
6. **Cross-posting massif identique** : même titre + même body partout (HN, Reddit, Twitter, LinkedIn). Reddit + HN détectent et déclassent.
7. **Reposter HN après échec premier launch** : HN pénalise les reposts du même URL. Si tu floppes, attends 6+ mois et re-frame complètement.

## Mesure / KPIs

- **HN frontpage minutes** : time on frontpage = proxy direct du traffic (1h frontpage ≈ 5-10k visites).
- **PH ranking** : #1-3 Product of the Day = succès. Top 5 = correct. <#10 = échec relatif.
- **Sign-ups dans les 48h post-launch** : conversion réelle, pas vanity.
- **GitHub stars J+7 / J+30** pour OSS.
- **Inbound mentions** semaine post-launch (Twitter, podcasts, newsletters tech). Vraie traction = relais sans demander.
- Benchmarks 2026 : Show HN frontpage 4h = 5-15k visites + 100-500 sign-ups si CTA clair. PH #1 = 2-5k visites + 200-1000 sign-ups.
